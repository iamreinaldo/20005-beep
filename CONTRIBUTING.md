# Contributing to 20-005 Beep

Obrigado pelo interesse em contribuir com o projeto.

O objetivo do 20-005 Beep é construir uma plataforma open source para centralização, organização e monitoramento de serviços self-hosted.

Toda contribuição é bem-vinda, seja código, documentação, testes, correções de bugs ou sugestões de funcionalidades.

---

## Como contribuir

1. Faça um fork do repositório.
2. Crie uma branch para sua alteração.
3. Implemente suas mudanças.
4. Teste antes de enviar.
5. Abra um Pull Request descrevendo claramente o que foi alterado.

Exemplo:

```bash
git checkout -b feature/service-monitoring
```

---

## Padrões do projeto

### Código

- Utilize nomes claros e descritivos.
- Evite código morto ou não utilizado.
- Mantenha funções pequenas e focadas em uma única responsabilidade.
- Sempre que possível, documente comportamentos não óbvios.

### Backend

- FastAPI
- SQLAlchemy
- Pydantic
- PostgreSQL

### Frontend

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui

---

## Relatando bugs

Ao abrir uma Issue, informe:

- O comportamento esperado
- O comportamento observado
- Passos para reproduzir o problema
- Logs ou mensagens de erro relevantes
- Ambiente utilizado

---

## Sugestão de funcionalidades

Antes de iniciar uma implementação grande, abra uma Issue para discussão.

Isso evita trabalho duplicado e ajuda a manter o alinhamento do projeto.

---

## Pull Requests

Pull Requests devem:

- Resolver um problema específico
- Ter descrição clara
- Não incluir alterações não relacionadas
- Manter compatibilidade com funcionalidades existentes

---

## Código de Conduta

Todos os participantes devem manter um ambiente respeitoso e colaborativo.

Discussões técnicas são encorajadas. Ataques pessoais não serão tolerados.

---

Obrigado por ajudar a construir o 20-005 Beep.
